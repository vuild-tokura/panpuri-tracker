# ガントチャート & フェーズベースタブ再編 設計書

## 概要

什器製作管理シート（panpuri tracker）のタブ構成をフェーズベースに再編し、ガントチャートによる全体俯瞰ビューを追加する。

## 背景・課題

- 現在のタブ構成（品番一覧/仕上げ/部品/承認/CNC/フェーズ/ToDo）はデータ種別ベースで、「どこでスタックしているか」「誰（どの会社）が担当か」が一目でわかりづらい
- 最終更新日時が記録されておらず、データの鮮度が不明
- プロジェクト期間: 2026-03-23 〜 2026-04-16（約3.5週間）

## タブ構成（9タブ）

| タブ | データソース | 詳細度 |
|------|------------|--------|
| **ガント** | 新規 | 品番×7フェーズのガントチャート（全体俯瞰） |
| **材料調達** | 既存部品タブから移行 | 品番・部品・素材・板厚・数量・発注状況・会社・画像・備考 |
| **木部CNC加工** | 既存CNCタブから移行 | 品番・部品・加工種別・データステータス・CNCステータス・担当・画像 |
| **仕上げ** | 既存仕上げ+承認を統合 | 品番・部品・仕上げ仕様・サンプル確認・承認ステータス・画像 |
| **金属加工** | 新規（シンプル） | 品番・ステータス・担当会社・備考 |
| **組立** | 新規（シンプル） | 品番・ステータス・担当会社・備考 |
| **検品** | 新規（シンプル） | 品番・ステータス・担当会社・備考 |
| **納品** | 新規（シンプル） | 品番・ステータス・担当会社・備考 |
| **ToDo** | 既存維持 | 各タブのTODO集約 |

### 廃止タブ
- 品番一覧 → ガントに統合
- フェーズ → ガントに統合
- 部品 → 材料調達に再編
- 承認 → 仕上げに統合
- CNC加工 → 木部CNC加工に再編

### 時間軸
- 範囲: 2026-03-23 〜 2026-04-16（25日間）
- 粒度: 日単位
- 今日ライン: 赤い縦線で今日の日付を強調

### バー色（ステータス連動）
| ステータス | 色 |
|-----------|-----|
| 未着手 | グレー |
| 進行中 | 青 |
| 完了 | 緑 |
| 保留 | オレンジ |

### 操作方法
- **期間設定（方法1）**: セルクリック → カレンダーピッカーで開始日・終了日を入力
- **期間設定（方法2）**: 日付セルをドラッグ選択 → その範囲が開始日〜終了日に設定
- **会社選択**: CycleBadge（クリックで切り替え、プリセットリストから選択）
- **ステータス変更**: CycleBadge（未着手→進行中→完了→保留）

### 自動ステータス連動
他タブの実データからガントのバー色が自動変更される:

| フェーズ | 参照データ | 連動ロジック |
|---------|----------|------------|
| 材料調達 | 材料調達タブの`procure` | 全品入荷済→緑、欠品あり→赤、未発注→グレー |
| 木部CNC加工 | CNCタブの`cncStatus` | 全完了→緑、進行中→青、未着手→グレー |
| 仕上げ | 仕上げタブの`status`+`sample` | 仕様未確認→赤、確認中→オレンジ、承認済→青/緑 |
| 金属加工 | ガントの手動ステータス | — |
| 組立 | ガントの手動ステータス | — |
| 検品 | ガントの手動ステータス | — |
| 納品 | ガントの手動ステータス | — |

## データモデル

### 新規テーブル: `gantt`

| カラム | 型 | 説明 |
|--------|-----|------|
| id | string | 自動生成ID |
| item | string | 品番（"LF-01"等） |
| phase | string | フェーズ名（7種） |
| company | string | 担当会社（プリセット） |
| start_date | string | 開始日（YYYY-MM-DD） |
| end_date | string | 終了日（YYYY-MM-DD） |
| status | string | 未着手/進行中/完了/保留 |
| note | string | 備考 |
| updated_at | string | 最終更新日時 |

### 新規シンプルテーブル: `metalwork`, `assembly`, `inspection`, `delivery`

| カラム | 型 | 説明 |
|--------|-----|------|
| id | string | 自動生成ID |
| item | string | 品番 |
| status | string | 未着手/進行中/完了/保留 |
| company | string | 担当会社（プリセット） |
| note | string | 備考 |
| updated_at | string | 最終更新日時 |

### 既存テーブル変更
- 全テーブル（items, finish, parts, approval, cnc, phase）に`updated_at`カラムを追加

### 会社プリセットリスト
`initialData.ts`にて定義（具体的な会社名は別途入力）

## ヘッダー
- 右上に全データの最終更新日時を表示
- 各行に小さくグレーで個別の更新日時を表示

## 技術的な注意事項
- 既存の`useSupabaseTable`フックを新テーブルにも適用
- ガントチャートのドラッグ選択はReactのマウスイベント（mousedown/mousemove/mouseup）で実装
- カレンダーピッカーはネイティブの`<input type="date">`を使用
- 品番フィルターは既存ロジックを流用
